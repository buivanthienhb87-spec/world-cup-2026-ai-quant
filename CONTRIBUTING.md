# Contributing

Thank you for considering a contribution. This project is intended to be a factual, maintainable, and community-friendly reference project. Contributions should improve clarity, data quality, documentation, examples, or reproducibility.

Before opening a pull request, please check that your change is relevant to the repository scope, uses respectful language, avoids misleading claims, and cites official or verifiable sources where appropriate. Promotional content, unsafe tooling, spam links, gambling content, cheating instructions, and unsupported claims will not be accepted.

## Pull request checklist

| Area | Requirement |
|---|---|
| Accuracy | Use official or verifiable references where factual claims are made. |
| Safety | Do not include credential leaks, malicious scripts, cheats, bypasses, or unsafe automation. |
| Maintainability | Prefer simple file structures and clear naming conventions. |
| Documentation | Update README or related docs when behavior or structure changes. |
| Licensing | Only contribute content that can be published under the project license. |
