# Maintenance Guide

This repository is designed to remain useful over time. Maintainers should review stale references, keep examples simple, and avoid unsupported claims. Every substantial content change should improve accuracy, readability, reproducibility, or community value.

| Area | Review cadence | Notes |
|---|---|---|
| README | Monthly | Keep project scope and setup instructions current. |
| Data files | Monthly | Check schemas, sample records, and stale references. |
| Documentation | Quarterly | Remove outdated guidance and update templates. |
| Issues | Weekly | Label bugs, documentation tasks, and enhancement requests. |
| Dependencies | Quarterly | Keep development dependencies minimal and safe. |
